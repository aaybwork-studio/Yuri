# Plan 01: Core Brand Foundations

Establish typography integration and core Tailwind CSS v4 variables as the foundation for the visual system.

## Tasks

### Task 1: Font Integration and Scroll Setup
* **<files>**: `layout/theme.liquid`
* **<action>**: Integrate Space Grotesk and Hanken Grotesk fonts into `layout/theme.liquid` using the Google Fonts API. Add preconnect and dns-prefetch hints for fonts.googleapis.com and fonts.gstatic.com (D-14). Load all weights from 300 to 800 and italics (D-15). Preload critical weights of Space Grotesk (Regular/700) and Hanken Grotesk (Light/Regular) (D-17) to minimize unstyled text flashes (FOUT). Add a simple client-side vanilla JavaScript scroll listener to toggle an `is-scrolled` class on the `.header-wrapper` container when scroll position is greater than 0, enabling the transparent-to-solid background transition (D-06). Avoid loading local font files to optimize caching, and do not introduce jQuery or heavy scroll libraries.
* **<verify>**: `shopify theme check`
* **<done>**: Google Font preconnects, preloads, and stylesheet link tags are present in the layout head, and the scroll state toggle script is added successfully.

### Task 2: Tailwind CSS v4 Variables & Resets Configuration
* **<files>**: `frontend/entry.css`
* **<action>**: Update the `@theme` block in `frontend/entry.css` to define YURI's brand semantic variables: `--color-brand-bg` for `#eeecd3`, `--color-brand-text` for `#403002`, `--color-accent-rose` for `#cc1574`, and `--color-accent-gold` for `#efbf04` (D-01). Declare font families `--font-serif` ("Space Grotesk") and `--font-sans` ("Hanken Grotesk"). Apply global CSS overrides forcing Space Grotesk on headings and Hanken Grotesk on body (D-10). Enable global font-smoothing resets (`-webkit-font-smoothing: antialiased`, `text-rendering: optimizeLegibility`) (D-16). Style the body background as flat `#eeecd3` (D-03). Override borders and dividers (like `.card`, `.card-wrapper`, `.utility-bar`, and custom grid dividers) to use a transparent mix of the primary text color (`#403002` at 10% opacity) (D-04). Ensure card backgrounds are transparent/borderless (D-03). Add styles for the navigation header wrapper `.header-wrapper` in `frontend/entry.css` to transition from transparent (when overlaying the slideshow at the top of the index page) to solid `#eeecd3` with a thin border (using 10% opacity primary text color) when the `.is-scrolled` class is present (D-06). Avoid using the deprecated tailwind.config.js, as Tailwind v4 is configured via CSS.
* **<verify>**: `npm run build`
* **<done>**: `frontend/entry.css` contains all brand-new color, font, and reset declarations, and compiles into `assets/yuri-styles.css`.

## Threat Model

### Trust Boundaries
- Client-side browser (executes scripts, loads stylesheets/fonts)
- External CDN (Google Fonts servers)
- Vite dev server (compiles entry.css)

### STRIDE Register
- **Spoofing**: Injection of a malicious external stylesheet/font link. Mitigation: Use official Google Fonts URLs.
- **Tampering**: Manipulation of compiled CSS assets. Mitigation: Control write permissions in the local repository and verify the Vite compilation output.
- **Repudiation**: Untracked modifications of brand variables. Mitigation: Commit all configurations to git and track changes.
- **Information Disclosure**: Leakage of theme editor details. Mitigation: Ensure no API keys or access tokens are written into theme files.
- **Denial of Service**: Excessive or blocking font requests stalling page loading. Mitigation: Preload only the core weights (Space Grotesk Regular/700, Hanken Grotesk Light/Regular).
- **Elevation of Privilege**: None (CSS and theme layout code have standard client-side browser privilege limitations).
